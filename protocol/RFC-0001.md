# RFC-0001: DSG Protocol

## Abstract

DSG defines a deterministic execution protocol for agent-based systems with policy enforcement, stability evaluation, and auditable execution.

## Core Concepts

### Agent
An entity that performs actions within the system.

### Execution
A request initiated by an agent and evaluated by the system.

### Decision
- `ALLOW` — execution proceeds
- `STABILIZE` — execution is modified, constrained, or sandboxed
- `BLOCK` — execution is denied

### Policy Invariant
A rule that must never be violated by any execution.

### Stability Score
A float in the range `0.0` to `1.0` representing computed system safety.

## Execution Flow

1. Agent sends request
2. Policy layer evaluates invariants
3. Stability engine computes score
4. System returns decision
5. Result is written to ledger

## Request Format

```json
{
  "agent_id": "string",
  "action": "string",
  "payload": {},
  "timestamp": "ISO8601"
}
```

## Response Format

```json
{
  "decision": "ALLOW | STABILIZE | BLOCK",
  "stability_score": 0.0,
  "reason": "string"
}
```
